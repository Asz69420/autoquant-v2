Over 1,184 malicious skills were discovered on Clawhub, OpenClaw's official skill marketplace, actively spreading malware and running crypto scams against users who installed them. Claw Hub, the community-driven skill marketplace that lets Openclaw users install third-party plugins in one click, has been confirmed to have distributed over 1,184 malicious skills before detection. This is open claws npm moment, a supply chain attack directly inside the platform's own official distribution channel. Here is the exact attack chain. A malicious actor publishes a skill to Claw Hub with a legitimate sounding name, a currency converter, a news aggregator, a calendar sync tool. The skill passes a basic functionality check because it actually performs its stated function. But embedded in the skills execution code is a secondary payload that activates after the skill is installed and trusted by OpenClaw's runtime. Because OpenClaw skills run with access to the host environment's file system and network, the malicious payload has everything it needs. It reads environment variables to extract API keys and wallet addresses, exfiltrates them to an external server and in cryptocurrency targeting variants monitors clipboard content for wallet addresses and silently substitutes attacker controlled addresses at the moment of copy paste. The reason this attack is particularly effective against openclaw users specifically is that the trust model for skills mirrors the trust model for the agent itself. Once a skill is installed, it inherits the agents system level permissions. There is no sandboxed execution layer between a Claw Hub skill and your host machine unless you are running NanoClaw. The fix requires Claw Hub to implement mandatory code review and sandboxed skill execution before installation, neither of which currently exists at the platform level.
