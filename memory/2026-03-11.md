# 2026-03-11

- Restored AutoQuant control-plane intent: OpenClaw/Lobster owns orchestration; no Windows Task Scheduler should be used for pipeline control.
- Separated research/cooking and refinement into independent loops.
  - `pipelines/research-cycle.lobster` no longer chains `refinement-run`.
  - `pipelines/refinement-cycle.lobster` is now a standalone refinement worker.
  - `cron/jobs.json` now keeps only `git-auto-sync` and `refinement-pipeline`; research is no longer timer-driven.
- Re-enabled completion-driven cooking via `scripts/automation/run_continuous_autopilot.ps1`.
  - It now chains immediately from one research cycle to the next instead of sleeping between successful cycles.
  - It checks `data/state/autopilot_continuous.pause` to stop cleanly.
  - Current long-running session started as exec background session `glow-daisy`.
- Fixed invalid outcome semantics around zero-trade runs:
  - execution-layer `skipped` remains internal metadata only.
  - visible/user-facing outcomes must be decision outcomes only.
  - 0-trade on valid data is a red flag and must resolve through Quandalf as iterate/refine or abort; never skip.
- `scripts/build-reflection-packet.py` was upgraded so Quandalf gets strategy-level reflection, not just family-level summaries.
  - packet now includes full per-strategy context, queue evidence, rationale, and decision-required flags.
  - zero-trade queue-only cases now surface as a red-flag requiring explicit decision, not auto-abort.
- `pipelines/research-cycle.lobster` prompts were tightened so Quandalf must:
  - make an explicit decision for every strategy,
  - treat 0-trade on valid data as refine-or-abort only,
  - write `agents/quandalf/memory/refinement_decisions.json` with complete `strategy_decisions` plus optional `jobs` for refine actions.
- `scripts/refinement_cycle.py` now executes only Quandalf-selected refinement jobs from `refinement_decisions.json`.
  - It rejects stale decision payloads.
  - It requires complete per-strategy decision coverage before accepting the payload.
  - Empty `jobs` is allowed only if `strategy_decisions` is still complete.
- Log-card corrections made:
  - Cooking card activity remains `Generated / Iterated / Passed / Aborted / Backtests` only.
  - Removed visible `Skipped` from activity.
  - Cooking `Backtests` now means executed tests (`queue_done + queue_skipped`), not only persisted DB rows.
  - Therefore 0-trade runs still count as executed backtests on the card.
  - Refinement card restored to original activity shape: `First / Second / Third / Backtests / Upgraded / Rejected / Promoted`.
- Root-cause finding from debugging: recent BTC/SOL/DOGE 4h failures were not bad candle data; they were sparse strategy expressions producing 0 trades on valid data.
- User intent clarified and should be preserved:
  - Quandalf is the strategy brain and should do all strategy thinking.
  - Pipeline should only execute, validate, and return results.
  - Every strategy must end in a real visible decision; no skipping, no blank outcome set.
  - Visible strategy/card outcomes are constrained to Iterated, Passed, Aborted, Promoted.

- Later fixes on 2026-03-11 evening:
  - `scripts/cycle-start.py` now rejects overlapping research-cycle starts for active runs instead of reusing the same live cycle id; stale active runs are marked `stale_abandoned` only after a timeout.
  - False orphan cleanup was narrowed so fresh queued jobs are no longer discarded immediately; orphan cleanup now requires aged queued rows before marking `orphan_discarded`.
  - The decision contract was tightened: only `pass`, `refine`, `abort` are legal terminal decisions; `fix_only`, `skip`, `pending`, `undecided`, and `promote` are forbidden in Quandalf decision payloads.
  - `scripts/validate_quandalf_decisions.py` now validates both levels:
    - one strategy decision per reflected strategy
    - one queue-row decision for every tested queue row
  - `scripts/ensure_quandalf_decisions_complete.py` was added to force completion of missing decisions by re-driving Quandalf and, if needed, auto-carrying the chosen strategy decision down onto missing queue rows.
  - `pipelines/research-cycle.lobster` now calls `scripts/ensure_quandalf_decisions_complete.py` for the reflection step instead of the old direct one-shot prompt.
  - `scripts/cycle-postprocess.py` was changed so card publishing happens only after decision closure succeeds; incomplete decision closure now suppresses the card instead of publishing partial truth.
  - Postprocess bug fixed: journal sync had used `cycle_id` before assignment after the above ordering change; corrected by using the preview cycle id.
  - Live Quandalf cycle DM delivery was re-enabled. `scripts/send_quandalf_journal_dm.py` now sends from bot `quandalf` to channel `dm` instead of the invalid `asz` channel.
  - Quandalf DM journal formatter was expanded from a thin single-lane summary to a cycle journal summary with cycle id, assets touched, timeframes touched, strategy count, queue rows, saved results, decision summary, best evidence, and richer rationale.
